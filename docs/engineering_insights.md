# 从一次复现中体现出的工程思考

## 1. 版本不是列表，而是一张约束图

Python、PyTorch、CUDA、Isaac Gym、CSDF 与 PointNet2 之间存在 ABI 和 API 约束。有效的环境管理不是逐个安装最新版，而是找出一组同时满足约束的组合。

## 2. 把复杂系统拆成可证伪的层

我使用“解释器 → 框架 → CUDA 扩展 → 仿真器 → 资产 → task → PPO”的顺序。每层都有最小测试，因此出现错误时可以说明哪一层已经被排除。

## 3. 复现工作的价值不只是跑通

真正可迁移的成果包括：兼容矩阵、失败树、自动检查脚本、可复用的训练/测试入口以及准确的验证边界。这些能降低下一次复现同类机器人学习项目的成本。

## 4. PPO 与机器人任务不是两套知识

PPO 的 observation、action、reward、done 都由仿真任务定义。关节顺序、动作缩放、接触状态或坐标系错误会直接改变 MDP，即使优化器代码完全正确也学不到目标行为。

## 5. 对结果保持工程诚信

运行成功、任务行为成功和论文指标复现是不同层级。本项目只声明有证据支持的“核心工程流程复现”，并把待补的曲线、视频与定量评估单独列出。
