# 这次复现带给我的工程认识

## 1. 版本不是一张安装清单

我最初逐项安装依赖，遇到冲突就单独换版本。后来才发现 Python、PyTorch、CUDA、Isaac Gym、CSDF 和 PointNet2 组成的是约束图。某个包单独安装成功，不说明整套组合可以工作。

## 2. 新报错不一定意味着方向错了

排错过程中经常出现一个错误消失、下一个错误马上出现。最开始我会怀疑前面的修改没有作用。实际观察调用栈后，我发现这通常说明程序终于进入了下一层。

## 3. 文件存在不等于语义一致

数据 key、关节数量、关节顺序、坐标系和 scale 都属于 schema。只检查路径或文件名容易把不同机器人数据错误地连接起来。

## 4. PPO 与 task 不能分开理解

observation、action、reward 和 reset 共同定义 MDP。机器人自由度或动作缩放错误时，修改 PPO 超参数无法解决问题。

## 5. 我需要区分不同层级的“成功”

- 环境成功：依赖和 Isaac Gym example 可以运行。
- 程序成功：generation、训练、测试和可视化入口能够执行。
- 任务成功：策略产生符合目标的行为。
- 论文复现：在相同数据与协议下获得可比较指标。

本仓库目前只声明有证据支持的核心工程流程复现。
